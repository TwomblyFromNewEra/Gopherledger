# Gopherledger

Это бэкенд системы лояльности для интернет-магазина. Суть простая: пользователи регистрируются, загружают номера своих покупок и получают за них бонусные баллы. Накопленными баллами можно частично оплачивать следующие заказы.

Проект писал с фокусом на глубокое понимание языка — здесь по максимуму используется **стандартная библиотека Go**, без тяжелых фреймворков и сторонних ORM.

## Что под капотом (Технические детали)

* **Роутинг и API:** Написаны на стандартном `net/http`. Реализовал свои мидлвари для логирования, обработки паник (Recover) и проверки токенов авторизации.
* **Хранилище:** In-memory база данных на мапах. Для потокобезопасности везде использую `sync.RWMutex`. Списание баллов (Withdraw) сделано атомарным, чтобы баланс ни при каких условиях гонок (race conditions) не ушел в минус.
* **Фоновые задачи:** Реализовал паттерн Worker Pool. При старте сервера запускается фоновый воркер, который асинхронно разгребает новые заказы и начисляет баллы с ограничением по количеству одновременно работающих горутин.
* **Алгоритмы:** Номера заказов при загрузке валидируются по алгоритму Луна.
* **Безопасность:** Пароли хешируются через `crypto/sha256`. Авторизация работает по самописным токенам.
* **Тесты:** Написал табличные unit-тесты для бизнес-логики и интеграционные тесты для HTTP-ручек (через `net/http/httptest`). Тестовое покрытие (coverage) > 40%.

## Как это работает

1. Пользователь регается и получает токен (передается в заголовке `Authorization`).
2. Закидывает номер заказа. Заказ попадает в статус `NEW`.
3. Фоновый воркер берет заказ в работу (`PROCESSING`), имитирует проверку и либо начисляет баллы (`PROCESSED`), либо бракует заказ (`INVALID`).
4. Пользователь проверяет баланс и может списать баллы в счет новой покупки.
